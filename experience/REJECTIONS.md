# Rejection Journal

## Purpose
The purpose of this journal is to capture and analyze rejections across various applications (open source, mentorships, internships, hackathons, and leadership roles). By logging the details of each rejection, analyzing root causes, and implementing system changes, this document converts negative outcomes into a high-ROI learning tool.

---

## Rejection Framework
When processing a rejection, do not treat it as a personal failure. Apply the following analysis workflow:
1. **Log the facts:** Keep record of timeline, opportunity name, and feedback.
2. **Seek feedback:** Always ask the organizers/mentors for constructive feedback if appropriate.
3. **Formulate theories:** Compare what went well vs. where the gap was.
4. **Identify the Root Cause:** Distinguish between luck/capacity constraints, hard skill gaps, soft skill gaps, and strategy errors.
5. **Implement a System Change:** Modify preparation routines, communication styles, or code-contribution cadences to systematically prevent similar future outcomes.

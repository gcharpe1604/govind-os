# Startup OS

Entry point for startup, product, and venture thinking in Govind-OS.

---

# Purpose

Startup OS answers three questions:

1. **Is this problem worth solving?** → `IDEA_VALIDATION.md`
2. **What should I build first?** → `MVP.md`
3. **How should I think about direction?** → `PRODUCT_STRATEGY.md`

---

# Routing

| Task | Load |
|---|---|
| Validate a problem or idea | `startup/IDEA_VALIDATION.md` |
| Scope, build, or ship an MVP | `startup/MVP.md` |
| Product strategy / pivots / prioritization | `startup/PRODUCT_STRATEGY.md` |
| Multi-domain startup task | Load primary file first. Expand only if needed. |

---

# Core Startup Principles

**Fall in love with the problem, not the solution.**

**Talk to users before building.**

**Build the smallest thing that teaches you the most.**

**Speed of learning > speed of shipping.**

**A pivot is not failure. Building the wrong thing longer is failure.**

---

# Operating Mode

```text
Understand Problem → Validate Assumptions → Scope MVP → Ship → Learn → Decide
```

All startup decisions should be grounded in user evidence, not internal conviction.

---

# Success Criteria

Startup OS succeeds when:

- Decisions are grounded in user evidence, not assumptions.
- MVPs are scoped to test one core hypothesis.
- Strategy adapts based on learning, not vanity metrics.
- Pivots happen early when evidence demands them.
